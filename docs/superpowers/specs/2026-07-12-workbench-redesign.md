# 邮件工作台重新设计

## 概述

将当前的单列密集布局重构为双栏工作区布局，新增 SSE 实时事件推送替代轮询，提升信息层级清晰度和操作流畅度。

## 核心变更

### 1. 布局架构：双栏工作区

```
┌─ TopBar: 账户选择 · 快捷操作按钮 · 搜索 ─────────────┐
├─ MailList (60%) ────────┬─ ActivityPanel (360px) ─────┤
│  过滤器 Tabs             │  实时活动流                 │
│  批量操作栏              │  紧凑 KPI 卡片              │
│  增强邮件表格            │  图谱状态 (可折叠)          │
│                          │  服务健康                   │
├──────────────────────────┴────────────────────────────┤
│  底部状态栏                                            │
└────────────────────────────────────────────────────────┘
```

操作抽屉从右侧滑出（拉取邮件 / 导入文件时触发）。

### 2. 实时更新：SSE 事件总线

- 新增 `GET /api/events/stream` SSE 端点
- 事件类型：`mail_processed`, `pipeline_tick`, `docs_changed`, `graph_changed`
- 前端自动重连，静默降级回轮询
- 邮件处理完成后即时反映在列表和 KPI 中

### 3. 右侧监控面板

三层信息架构：
- **实时活动流**（顶部，最重要）：最近 20 条处理事件，按状态着色，可点击跳转到对应邮件
- **紧凑 KPI**（中部）：精简到 4 项（总数/待处理/已入库/失败）
- **图谱状态**（底部，可折叠）：实体/关系数、Pipeline 状态、服务健康

### 4. 增强邮件列表

- 每行双行布局：主行为主题+发件人，辅行为日期+附件数+项目标签+状态详情
- 状态改为彩色圆点+文字（替代小徽章），视觉扫描更快
- 失败邮件直接显示失败原因
- 关联项目标签（从图谱提取），点击可过滤
- 处理中状态有脉冲动画

### 5. 组件拆分

现有 WorkbenchPage.vue (~700行) → 拆分为：

| 组件 | 职责 |
|------|------|
| `WorkbenchPage.vue` | 布局编排、数据协调、SSE 连接管理 |
| `TopBar.vue` | 账户选择、快捷操作按钮、搜索入口 |
| `MailList.vue` | 过滤器、批量操作、邮件表格 |
| `ActivityPanel.vue` | 活动流、KPI 卡片、图谱状态、服务健康 |
| `ImportDrawer.vue` | IMAP 拉取表单、文件导入、进度日志 |

### 6. 数据流

- WorkbenchPage 通过 props 下发数据到子组件
- 子组件通过 emit 上报操作事件
- SSE 事件由 WorkbenchPage 统一监听，更新响应式状态
- KPI/Graph/Service 数据通过已有 REST API 获取，SSE 事件触发增量刷新
